## Vite `base` 路径（必看）

当前面向 **GitHub Pages 项目站**（路径为 `/仓库名/`），`vite.config.ts` 中已设置：

```ts
base: '/WYBOT-APP-DEMO/',
```

| 场景 | `base` 建议 |
|------|----------------|
| GitHub Pages：`username.github.io/仓库名/` | `'/仓库名/'`（与仓库名一致） |
| 根域名 / Vercel / Netlify 根路径 | `'/'` |
| 子路径但仓库改名 | 同步修改 `base` 为 `'/新仓库名/'` |

改错会导致页面白屏、图片/JS/CSS 404。

## 静态资源（图片）引用方式

**不要**在代码里写 `src="/src/assets/..."` 或 `url(/src/assets/...)`。开发时 Vite 会提供 `/src`，但 **`npm run build` 后没有该路径**，线上会 404，表现为部分图片不显示（本地 Windows 有时仍“看起来正常”）。

正确做法：

- 在脚本里 **`import img from '../assets/xxx.png'`**，再在 `src={img}` 或 `` style={{ backgroundImage: `url(${img})` }} `` 中使用；或
- 将固定资源放在 **`public/`** 下，用绝对路径 `"/文件名"`（需带仓库前缀时写成 `import.meta.env.BASE_URL + '文件名'`）。

## GitHub 一次性设置

1. 仓库 **Settings → Pages**
2. **Build and deployment** 中 **Source** 选 **GitHub Actions**（不要选 “Deploy from a branch”）
3. 首次可在 **Actions** 里手动运行 **Deploy to GitHub Pages**

---

## 日常发布流程

1. 本地开发与自测：`npm run dev`
2. 打包自检：`npm run build`，必要时 `npm run preview`
3. 提交并推送 `main` 分支：

   ```bash
   git add .
   git commit -m "说明本次修改"
   git push origin main
   ```

4. 在 **Actions** 中确认工作流成功；几分钟后访问线上地址验证。

工作流文件：`.github/workflows/deploy.yml`（构建命令为 `npm ci` + `npm run build`，发布目录为 `dist`）。

---

## 若迁到其他平台

- 若改为 **Vercel / Netlify / 自有域名根路径**：将 `base` 改回 `'/'`，并检查该平台构建命令与输出目录（一般为 `npm run build` → `dist`）。
- 若仍用 **GitHub Pages** 但 **仓库改名**：必须同步修改 `vite.config.ts` 里的 `base` 与仓库名一致。
